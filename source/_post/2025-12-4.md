# 全自动！我在三轮车上按了一个按钮，我的个人博客就更新了（附手机端 CI/CD 方案）

![](请插入一张你的手机屏幕特写，屏幕上显示着 Vercel 的 "Deployment Successful" 或者 MarkdownWeaver 的界面)

我是庹林。

就在刚刚，我送完一单辅料，把三轮车停在路边。
我拿出那台碎屏的三星手机，打开我自己开发的 **MarkdownWeaver**，写下了几句感悟，然后点击了一个新加的按钮——**“Push to Blog”**。

30 秒后，手机震动了一下。
**Vercel 通知：构建成功。**

我没有打开电脑，没有敲复杂的命令行，没有挂 VPN。
但我部署在 Vercel 上的个人博客 **`laotuo.top`**，已经自动更新了。

这一刻，我觉得自己不是一个困在株洲街头的送货员。
我是一个坐在云端指挥千军万马的**架构师**。

### 01. 痛点：被切断的“网线”

前几天，我虽然做出了 MarkdownWeaver，但它只是一个“排版工具”。
我要更新博客，还得回家打开电脑，翻墙，输命令……这对我这个每天在路上跑的人来说，太奢侈了。

而且，Vercel 的默认域名在国内访问不稳定。我的博客像一座孤岛，没人能看见。

**“Runtime Exception”再次抛出。**
但我这次没有慌。

### 02. 重构：打通任督二脉

我问 Gemini：“如果不依赖电脑，如何在手机上实现 **CI/CD（持续集成与部署）**？”
Gemini 给了我方案，我负责执行。

在送货的间隙，我重构了 MarkdownWeaver 的底层逻辑，给它装上了一颗**“Git 心脏”**。

**现在的 V2.0 版本，实现了什么？**

1.  **移动端 Git 集成：**
    我在 App 里内嵌了 Git 协议。现在，它能直接通过 API 把 `.md` 文件推送到 GitHub 仓库。
    *(手机 App -> GitHub API -> Vercel 触发构建 -> 网站更新)*

2.  **博客复活 (`laotuo.top`)：**
    通过技术手段（优选 IP 解析），我成功让博客在国内**免翻墙直连**。
    现在，无论你在哪里，输入 **`laotuo.top`**，就能看到我的最新动态。

### 03. 意义：手机即服务器

为什么要折腾这个？

为了**“自由”**。
为了证明：**即使身陷淤泥，只要技术在手，我就拥有向世界广播的权力。**
正如我博客上的那句话：**“生如逆旅，一苇以航。”**

现在的我，三轮车是我的工位，手机是我的服务器。
我想写什么，拿起手机就写；想发什么，点一下就发。
**不需要电脑，不需要环境，不需要等待。**

这种**“掌控感”**，是 AI 和技术赋予我们普通人最大的尊严。
